# Contributing to Paver Project

Thanks for your interest in contributing. This document describes how to report issues and contribute changes.

## Report issues

- Open an issue describing the problem, steps to reproduce, and expected vs actual behavior.
- Include Node.js version and OS when relevant.

## Propose changes (PRs)

1. Fork the repository and create a topic branch.
2. Make small, focused changes with clear commit messages.
3. Push your branch and open a pull request describing the change.

## Development setup

1. Install dependencies:

```bash
npm install
```

2. Run the app locally:

```bash
npm start
```

There are no automated tests included in this project; run manual verification against `index.html` and `admin.html`.

## Code style

- Follow existing project style (vanilla JavaScript + Node.js).
- Keep changes minimal and backwards-compatible.

## Questions

If you're unsure about a change, open an issue first to discuss.
